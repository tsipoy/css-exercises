Transitions exercises

  1. The `transition` property is a 'shorthand property'. What other properties are included in this shorthand property?
    There are: trasition-properties, transition-duration, transiton-delay, transition-timing-function.
  2. The difference between a timing function, delay, and duration. timing function defines how intermediate values are computed, delay defines how long to wait between the time of property is changed and the transition actually (mdn), and duration defines the duration over which transitions should occur (mdn).
  3. CSS properties animatable using transitions: width, height, background-color, color, font-size and background.
  4. Can we select more than one property to transition at a time?
    Yes, we can. For example, when we set the transition delay we can set the duration so we will know where it will be stopped and by setting timming function we can make how quick or slow it is.
  5. We set or define the desired transition state, in css by using :hover.
  6. Do we need transitions? What makes them useful?
    We do need transition when we want to animate properties or to make our properties be good looking instead of making it as simple.
  7. At what point can transitions become a problem?
    When we set our width as auto.
  8. Transitions don't work reliably with the `auto` keyword being one of the end states. Why is that?
    It is because the width is set automatically and if we define the transition-duration it create vibration until it reaches the value that we set.
  9. How can we stagger transitions? i.e. make a number of child elements all start at different times?
    We can use different transition-delay in different properties.
